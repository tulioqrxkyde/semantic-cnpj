PORTAL SEMÂNTICO DA SEFAZ-MA

Arquitetura da Plataforma Semântica da SEFAZ-MA



A Plataforma fornece um conjunto de componentes para Armazenamento, Acesso, Integração, Consulta e Gerenciamento dos dados da SEFAZ-MA . A Figura abaixo mostra os principais componentes da plataforma agrupados em cinco camadas:


1. Camada das Fontes de Dados: Na camada das fontes de dados estão armazenados os dados de todas as fontes que terão seus dados integrados. O formato de armazenamento destas fontes pode estar em diferentes formatos, e.g., bancos de dados relacionais, CSV, triple stores RDF, documentos JSON, etc. Vale destacar que uma fonte de dados pode ser composta de várias outras fontes, como o caso dos dados da RFB, que estão armazenados em 20 arquivos.

2. Camada Relacional: A Camada Relacional publica Visões Relacionais homogêneas que integram as várias fontes de Dados da SEFAZ-MA. As visões relacionais usam um vocabulário comum, o mesmo vocabulário da Ontologia de Domínio. As Visões Relacionais podem ser virtuais ou materializadas.

3. Camada Semântica ( Grafo de conhecimento Corporativo ): Na camada semântica, um grafo de conhecimento para cada fonte de dados é instanciado a partir de dados da camada relacional e usando o vocabulário da ontologia de Domínio. O grafo de conhecimento para um dada fonte local, pode ser virtual ou materializado.
Nessa camada, também são construídas ligações semânticas entre diferentes grafos de conhecimento, permitindo assim, o fluxo de informação entre os KGs das fontes de dados individuais.
O Grafo resultante da integração dos KGs individuais resulta no que denominamos de Grafo de Conhecimento Corporativo ( EKG : Enterprise Knowledge Graph).

4. Camada de Acesso e Integração de Dados: A principal função da camada de acesso é permitir o acesso integrado aos dados usando a ontologia de Domínio. Portanto, os usuários e aplicações podem realizar consultas de forma transparente, sem ter que entender sobre as fontes de Dados e estruturas de dados.

5. Camada De Aplicações : A camada de aplicações irá apresentar as interfaces necessárias para que usuários e aplicações acessem os metadados e dados via o EKG da SEFAZ/MA. A camada é composta de 3 módulos.

5.1 Portal Semântico: O objetivo do Portal Semântico é o Gerenciamento dos Metadados das 3 camadas de dados do EKG. Para representação desses metadados, propomos o vocabulário VEKG (referencia).
O Portal é constituído de 4 módulos:

  •   - Metadados para descrição camada das fontes de Dados
    •     • Esquema (relacional, CSV, etc)
    •     • Proveniência
    •     • Qualidade

    •   - Metadados para descrição da camada Relacional
    •     • Esquemas das Visões Relacionais ( virtuais ou Materializadas)
    •     • Consultas SQL p/ criação das visões Relacionais ( Mapeamentos SQL)
    •     • Especificação dos Links entre Visões Relacionais de diferentes fontes de dados

    •   - Metadados para descrição da camada semântica:
    •     • Ontologia de Domínio
    •     • Visão semântica Global
    •     • Visões Semânticas Locais (ontologia e mapeamentos)
    •     • Visões de Ligação



5.2. Aplicações Inteligentes Nesta camada podem ser utilizadas aplicações de consulta facetada, busca semântica, Q\&A como chatbots, sistemas de recomendação e outras aplicações de inteligência artificial e machine learning que utilizam da semântica para acessar o EKG da SEFAZ-MA através de um MS juntamente com um Federador SPARQL. Isso posto, tais aplicações podem usufruir da semântica provida através dos dados integrados no EKG da SEFAZ para realizar consultas de interesse no domínio de saúde.

  •   • Catálogo das aplicações
  •   • Criar Mashup de Dados Especializados (fusão e limpeza dos dados)
  •   • Consultas/ Aplicações Inteligentes
  •   • Gerenciar Catálogo de Consultas SPARQL



5.3. Interfaces para consulta, exploração e visualização : Permite aos usuários finais consultar, explorar e visualizar os dados da camada semântica e relacional por meio da Visão semântica (Ontologia de Domínio)

  •   • Consulta semântica baseada em palavra chave
  •   • Navegação interativa, exploração e visualização do EKG com base em conceitos relativos ao domínio da SEFAZ
  •   • Interface do Usuário (UI) intuitiva e amigável ao usuário final para descoberta, navegação, exploração e análise do conhecimento
  •   • Busca Semântica rica com utilização de facetas
